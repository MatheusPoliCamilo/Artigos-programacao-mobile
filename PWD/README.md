# Progressive Web Apps

### A web... mas melhor
Um **_Progressive Web App_ (PWA)** é um aplicativo Web que usa recursos modernos da **_Web_** para oferecer uma experiência semelhante a um 
aplicativo para os usuários. Esses aplicativos atendem a certos requisitos, são implantados em servidores, acessíveis por 
_**URLs**_ e indexados pelos mecanismos de pesquisa.

### O que é necessário

Para ser considerado um aplicativo Web progressivo, seu aplicativo deve ser:

* **Progressivo** - funciona para todos os usuários, independentemente da escolha do navegador, porque eles são criados com 
aprimoramento progressivo como princípio básico.

* **Responsivo** - Ajuste qualquer fator de forma, *desktop*, celular, *tablet* ou qualquer outro.

* **Independente de conectividade** - Aprimorado com *service workers* para trabalhar *offline* ou em redes de baixa qualidade.

* **Como aplicativo** - use o modelo de aplicativo **_shell_** para fornecer navegação e interações no estilo de aplicativo.

* **Recente** - Sempre atualizado graças ao processo de atualização do *service worker*.

* **Seguro** - Servido via **_HTTPS_** para evitar bisbilhoteiros e garantir que o conteúdo não foi violado.

* **Detectável** - são identificáveis como “aplicativos” graças aos manifestos do **W3C** e ao escopo de registro do *service worker*, 
permitindo que os mecanismos de pesquisa os encontrem.

* **Reengajável** - facilite o reengajamento por meio de recursos como notificações *push*.

* **Instalável** - Permite que os usuários “mantenham” os aplicativos que acharem mais úteis na tela inicial, sem a necessidade de uma 
loja de aplicativos.

* **Linkável** - compartilhe facilmente via *URL* e não exija instalação complexa.
